# RealPay Bot

Telegram-бот "магазин услуг" на aiogram 3 (SQLite).

## Файлы

- `bot.py` — весь код бота (конфиг, база данных, клавиатуры, все хендлеры, запуск).
- `requirements.txt` — зависимости Python.
- `.env.example` — пример файла с переменными окружения, скопируй в `.env` и заполни.
- `Procfile` — для деплоя на Heroku/Railway и подобные платформы (процесс типа worker, т.к. бот работает через long polling, не через веб-запросы).
- `.gitignore` — чтобы не закоммитить токен, базу и кеш Python.

## Возможности

- Приветствие и главное меню без эмодзи: **Купить услуги**, **Обратная связь**.
- Каталог: категории → услуги (название, цена, юзернейм) → карточка услуги
  с описанием и кнопкой **Написать**, ведущей на `t.me/username`.
- Обратная связь: сообщение пользователя пересылается главному админу
  (и админам с соответствующим правом). Ответ через *reply* на это
  сообщение автоматически уходит обратно пользователю.
- Админ-панель:
  - **Каталог** — добавление/удаление категорий и услуг.
  - **Тексты** — редактирование всех текстов бота.
  - **Обратная связь** — список писавших пользователей, удаление.
  - **Оформление** — смена подписи и эмодзи-маркера у любой кнопки.
  - **Управление админами** (только у главного админа) — добавление
    новых админов с ограниченным набором прав (`catalog`, `texts`,
    `feedback`, `appearance`), удаление админов.

## Ограничение платформы

Telegram Bot API не позволяет менять цвет фона inline-кнопок и вставлять
premium custom-эмодзи внутрь текста кнопки — это ограничение платформы,
а не бота. Вместо этого в разделе «Оформление» можно менять подпись и
обычный эмодзи-маркер перед текстом кнопки. Premium-эмодзи можно
вставлять в тексты сообщений через раздел «Тексты».

## Запуск локально

1. Установи зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Скопируй `.env.example` в `.env` и впиши свои значения:
   ```bash
   cp .env.example .env
   ```
3. Экспортируй переменные (или используй `python-dotenv`, если добавишь его
   сам — сейчас `bot.py` читает переменные окружения напрямую через `os.getenv`):
   ```bash
   export BOT_TOKEN="токен_от_BotFather"
   export OWNER_ID="твой_telegram_id"
   ```
4. Запусти:
   ```bash
   python bot.py
   ```

Свой Telegram ID можно узнать у бота @userinfobot.

## Деплой (Heroku / Railway и т.п.)

Используй `Procfile` — там указан процесс `worker`, так как бот работает
через long polling, а не принимает веб-запросы. На платформе укажи
переменные окружения `BOT_TOKEN` и `OWNER_ID` в настройках проекта
(Config Vars / Environment Variables), не через `.env`.

## База данных

SQLite-файл создаётся автоматически при первом запуске (`shop.db` по
умолчанию, путь можно переопределить переменной `DB_PATH`). Для деплоя на
платформы с эфемерной файловой системой (например, обычный Heroku dyno)
учти, что файл базы может сбрасываться при перезапуске — для продакшена
лучше подключить постоянное хранилище или внешнюю БД.
