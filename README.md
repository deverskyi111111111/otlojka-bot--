# RealPay Bot

Telegram-бот "магазин услуг" на aiogram 3 (SQLite).

## Файлы

- `bot.py` — весь код бота.
- `requirements.txt` — зависимости (нужен aiogram>=3.29.0 для поддержки цветных кнопок и premium-эмодзи, Bot API 9.4).
- `.env.example` — пример переменных окружения.
- `Procfile` — для деплоя (Heroku/Railway и т.п.), процесс типа worker (бот работает через long polling).
- `.gitignore`.

## Возможности

- Приветствие и главное меню: **Купить услуги**, **Обратная связь**.
- Каталог: preset-категории одним тапом (Telegram боты, Веб-сайты, Приложения, Дизайн, Копирайтинг, Хостинг, Парсеры) или своя категория → услуги (название, цена, юзернейм) → карточка с описанием и кнопкой **Написать** на `t.me/username`.
- Обратная связь: сообщение уходит главному админу (и админам с правом feedback), ответ через reply на пересланное сообщение уходит обратно пользователю.
- Рассылка всем, кто хоть раз писал боту.
- Статистика в админке: всего пользователей, новых за сегодня, категорий, услуг, админов, обращений, просмотров карточек услуг + топ-5 популярных услуг.
- Админ-панель с правами по разделам: `catalog`, `texts`, `feedback`, `appearance`, `stats` — можно выдавать по отдельности при добавлении нового админа.
- Редактирование любой услуги (цена/описание), не только удаление.
- **Оформление кнопок**: подпись, цвет (синий/зелёный/красный, через поле `style` из Bot API 9.4) и premium-эмодзи перед текстом (через `icon_custom_emoji_id`) — задаётся прямо в чате: перешли боту сообщение со своим premium-эмодзи, и он подставит его id.

## Про цвета и premium-эмодзи

Это стало возможно с Bot API 9.4 (9 февраля 2026): у `InlineKeyboardButton` появились поля `style` (`primary`/`success`/`danger`) и `icon_custom_emoji_id`. Для эмодзи нужно, чтобы **у владельца бота** (аккаунта, на который зарегистрирован токен) была подписка Telegram Premium — иначе Telegram эту иконку не покажет.

## Запуск локально

```bash
pip install -r requirements.txt
cp .env.example .env   # и заполнить
export BOT_TOKEN="токен_от_BotFather"
export OWNER_ID="твой_telegram_id"
python bot.py
```

## Деплой на Railway

1. Залей файлы в GitHub-репозиторий (без `.env`, он в `.gitignore`).
2. Railway → New Project → Deploy from GitHub repo.
3. В Variables добавь `BOT_TOKEN` и `OWNER_ID`.
4. Процесс `worker` из `Procfile` подхватится автоматически — HTTP-порт не нужен, бот работает через polling.
5. Для сохранения базы между деплоями подключи Volume и укажи `DB_PATH=/data/shop.db` (иначе `shop.db` может сбрасываться при пересборке).
